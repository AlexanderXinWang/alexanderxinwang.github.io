8 Ensemble Methods

Ensemble Methods

Recap: Decision Trees
Idea

Combine multiple weak learners - having either high bias or high variance - to create an ensemble with better prediction

Types
  • Combine multiple learners with high variance in a way that reduces their variance
  • Combine multiple learners with high bias in a way that reduces their bias
  • Parallel: build base estimators independently and then average their predictions. A combined estimator is usually better than any single base estimator because its variance is reduced.
  • Sequential: (boosting) build base estimators sequentially and each one tries to reduce the bias of the combined estimator.

Bagging

Background
  • Designed for, and most often applied to, decision trees
  • The name comes from bootstrap aggregation
Bootstrapping
  • Basic idea Sampling with replacement
  • Each “bootstrap training set” is the same size as a full training set, and is created by sampling with replacement
  • Some samples will appear more than once, and some samples not at all
Bootstrap Aggregation

  • Create multiple versions of a training set with bootstrap

  • Independently train a model on each bootstrap training set: calculate

  • Combine output of models by voting (classification) or averaging (regression):
    $$
    \hat{f}{bag}(x) = \frac{1}{B} \sum{b=1}^{B} \hat{f}_b(x)
    $$

Bagging Trees
  • Construct trees using bootstrapped training sets.
  • Let the trees grow deep, with no pruning.
  • Each individual tree has a low bias, high variance
  • Average the prediction of the trees to reduce variance
Correlated Trees

Problem: trees produced by bagging are highly correlated

  • Imagine there is one feature that is a strong predictor, several moderate predictors
  • Most/all trees will split on this feature
  • Averaging correlated quantities does not reduce variance as much
Random Forests

Grow many decorrelated trees:

  • Bootstrap: grow each tree with a bootstrap resampled data set
  • Split-variable randomization: Force each split to consider only a subset of m of the p predictors.

Typically but this should be considered a tuning parameter.

A Note on Computation
  • Bagged trees and random forests can be fitted in parallel on many cores
  • Each tree is built independently of the others

Boosting

Boosting - Training

Iteratively build a succession of models:

  • Train a weak model. Typically a very shallow tree.
  • In training set for th model, focuses on errors made by - th model
  • Use (weighted) model output
  • Reduces bias and variance
AdaBoost (Adaptive Boosting)

Adjust weights so that each successive model focuses on more “difficult” samples.

Gradient Boosting
  • The general goal of boosting: find the model at each stage that minimizes loss function on ensemble (computationally difficult)
  • AdaBoost interpretation (discovered a year later): Gradient descent algorithm that minimized exponential loss function.
  • Gradient boosting: works for any differentiable loss function. At each stage, find the local gradient of the loss function, and take steps in the direction of the steepest descent.

Summary

  • Can use a single estimator that has poor performance

  • Combining the output of multiple estimators into a single prediction: better predictive accuracy, less interpretability
